## What Is Phase 1?

Phase 1 covers the documents, role assignments, and evidence required before the policy suite can be trustee-approved and published. It spans three compliance areas: ADCH membership readiness, UK GDPR / ICO obligations, and Charity Commission governance expectations.

**2026-04-14 status summary:** The policy-design side of Phase 1 is substantively complete. Every animal-care and visits-and-volunteers policy has been revised to v2.0 against the ADCH Minimum Welfare Operational Standards, the supporting registers and templates are drafted, and a Forgotten Felines Emergency Plan is in place. Remaining Phase 1 work is principally **real-world data population** (vet arrangement letters, foster home register entries, fosterer training sign-off) and **three missing policies** (Safeguarding, Incident Reporting & Complaints, Fostering Agreement template).

## 5. Published Versions (.docx) — single authoritative source

All `.docx` versions of every policy, register, template, and the Phase 1 finalisation checklist live in **one folder**:

**[docs/forgotten-felines/published-policies/](../forgotten-felines/published-policies/)** — 33 files, refreshed from the current Markdown sources on 2026-04-14.

There are no other `.docx` copies in the repo. The previous duplicate `.docx` folder under `phase-1-deliverables/policies/` and the two stray files in the `phase-1-deliverables/` root have been removed to avoid version drift.

### Refreshing the .docx files

Whenever any Markdown source is updated, regenerate every `.docx` in one pass by running the converter from the repo root:

```
python3 scripts/md_to_docx.py
```

The script ([scripts/md_to_docx.py](../../scripts/md_to_docx.py)) walks every Markdown source listed in its `JOBS` table, parses headings / paragraphs / lists / tables / fenced code blocks / blockquotes / inline formatting, and writes the result to [published-policies/](../forgotten-felines/published-policies/). It is safe to run repeatedly — files are overwritten in place.
