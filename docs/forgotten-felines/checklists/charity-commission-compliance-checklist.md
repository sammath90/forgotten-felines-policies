## Phase 3: Serious Incident Readiness

> Trustees must be able to identify and report serious incidents promptly and correctly.

- [ ] Define and approve a **Serious Incident Reporting (SIR) procedure**
- [ ] Create an **incident log** covering safeguarding, financial loss, data breaches, and reputational incidents
- [ ] Define internal timeline for escalation to trustees (eg same-day for high-risk events)
- [ ] Define who submits SIRs and who reviews draft submissions before filing
- [ ] Add a checklist step to report to other regulators where relevant (ICO, police, local authority)
- [ ] Run one tabletop scenario to test SIR readiness
