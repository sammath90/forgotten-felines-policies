## Phase 4: Data Subject Rights Operations

- [ ] Define procedure and SLA tracking for **Subject Access Requests (SARs)**
- [ ] Define procedures for rectification, erasure, restriction, and objection requests
- [ ] Create a secure **rights request log** with decision and outcome evidence
- [ ] Provide trustee guidance on identity verification before disclosure
- [ ] Test one mock SAR end-to-end and capture lessons learned

---

## Phase 5: Security Controls & Breach Response

- [ ] Define a written **personal data breach response procedure**
- [ ] Ensure 72-hour ICO notification decision path is documented
- [ ] Define threshold for notifying affected individuals when high risk applies
- [ ] Restrict access to personal data on a least-privilege basis
- [ ] Require secure storage of signed acknowledgements and complaints logs
- [ ] Define rules for personal-device and messaging-app use for charity data

---

## Phase 6: Third Parties, Processors, and Communications Channels

- [ ] List all processors/third parties and their data-sharing purpose
- [ ] Put required data processing terms/contracts in place where applicable
- [ ] Define permitted vs high-risk channels for sensitive data
- [ ] Implement migration plan from consumer messaging for sensitive matters
- [ ] Document when to move a conversation from Messenger/WhatsApp to controlled channels

---

## Phase 7: Retention, Deletion, and Evidence

- [ ] Implement retention schedule operationally (not only as policy text)
- [ ] Define deletion methods and owner per record type
- [ ] Run periodic deletion checks and keep evidence
- [ ] Ensure backups/archives follow the same retention logic where possible
- [ ] Maintain an audit trail of compliance actions and policy approvals

---

## Phase 8: Training & Continuous Improvement

- [ ] Annual GDPR refresher for trustees and foster coordinators
- [ ] Onboarding GDPR briefing for new trustees/volunteers
- [ ] Annual policy review cycle (Trustee, Volunteer, Retention, Privacy Notice)
- [ ] Annual tabletop exercise for breach and SAR handling
- [ ] Quarterly review of incidents/near misses with corrective actions

---

## Summary — Current Priority Items

1. **Complete operational controls** missing behind existing GDPR policies (ROPA, rights workflow, breach procedure).  
2. **Publish and align privacy transparency** with actual processing and channels used.  
3. **Reduce channel risk** by defining and enforcing movement of sensitive data to controlled systems.
